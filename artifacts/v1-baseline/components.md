| Name | Description |
|---|---|
| `Accepted` | Request has been accepted for processing. |
| `BadRequest` | Bad Request |
| `Unauthorized` | Request lacks valid authentication credentials for the target resource |
| `NotFound` | The requested resource was not found. |
| `AccessDenied` | Authenticated participant request that violates some authorization rule. |
| `UnprocessableEntity` | Server understands the content type of the request entity, and the syntax of the request entity is correct, but it was unable to process the contained instructions. (Check if the instruction has not been sent before.) |
| `UnprocessableEntityOrderCancellation` | Server understands the content type of the request entity, and the syntax of the request entity is correct, but it was unable to process the contained instructions. (Check if the instruction has not been sent before.) |

| `priceMethod` | `string` | Instructions on how this `optionGroup` should be calculated before being applied to the item. **- SUM**: It will be added to the other optionGroups without any additional rules. **Default** option. **- MAX**: The highest `optionGroup` value between all `optionGroups` with `priceMethod` = `MAX` will be applied to the calculation. **- AVG**: The average between `optionGroup` with `priceMethod` = `AVG` will be applied to the calculation. Example: item base price: $25,00 optionGroup1 - priceMethod = **AVG** - __option 1 = $5,00 (selected)__ - option 2 = $10,00 (not selected) optionGroup2 - priceMethod = **AVG** - option 1 = $7,00 (not selected) - __option 2 = $10,00 (selected)__ optionGroup3 - priceMethod = **MAX** - __option 1 = $15,00 (selected)__ - option 2 = $20,00 (not selected) optionGroup4 - priceMethod = **MAX** - option 1 = $30,00 (not selected) - __option 2 = $35,00 (selected)__ optionGroup5 - priceMethod = **SUM** - option 1 = $2,00 (not selected) - __option 2 = $3,00 (selected)__ - option 3 = $5,00 (not selected) **optionsPrice** = avg(optionGroup1, optionGroup2) + max(optionGroup3, optionGroup4) + optionGroup5 **optionsPrice** = 7,50 + 35,00 + 3,00 **optionPrice** = 45,50 **itemPrice** = 25,00 + 45,50 = 70,50 |

### `ETA`

- Description: ETA

| Property | Type | Description |
|---|---|---|
| `updateMethod` | `string` | Update Method: Indicates how the times will be updated in the events. `OFFLINE`: The time will be calculated in the event `ACCEPTED` and will no longer be updated in the following events. `ONLINE`: The time will be updated in all events.
