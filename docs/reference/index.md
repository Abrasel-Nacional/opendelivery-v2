# Referência da API

Esta seção contém as especificações OpenAPI normativas para cada capability do Open Delivery V2.
Cada página renderiza o contrato REST completo — endpoints, schemas, exemplos de requisição/resposta e códigos de erro.

Para explicações conceituais, fluxos e orientações de implementação, consulte a seção **Protocolo**.

## Capabilities

| Capability | Descrição |
|---|---|
| [Autenticação](authentication.md) | Modelos OAuth 2.0, endpoint de token, assinatura de webhooks |
| [Discovery](discovery.md) | Endpoint well-known e declaração de capabilities |
| [Merchant](merchant.md) | Estabelecimento, catálogo, serviços e disponibilidade |
| [Pedidos](orders.md) | Ciclo de vida de pedidos, estados e eventos |
| [Logística](logistics.md) | Coordenação de entrega, rastreamento e problemas |
| [Customer & Fidelidade](customer.md) | CRM, leads, avaliações, programas de fidelidade e eventos |
| [Indoor](indoor.md) | Contas em salão, pedidos, pagamentos e fiscal |
