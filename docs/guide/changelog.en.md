## V2.0.0-rc — July 2026

First Release Candidate of Open Delivery Protocol V2. Result of six months of collaborative work
in technical committees with more than ten companies in the Brazilian food-tech ecosystem.

### Breaking changes

| Area | Change |
|---|---|
| **Authentication** | Auth by application — one `client_id` for all stores of the software, instead of one per store |
| **Merchant ID** | Generated by the originator (no longer by the POS). POS uses `externalCode` for internal correlation |
| **Menu** | Granular CRUD per entity replaces monolithic V1 `merchantUpdate` webhook |
| **Cancellation** | Mandatory handshake (request/accept/deny) removed. Direct simplified cancel |
| **Event `picked_up`** | Removed — ambiguous between takeout and logistics pickup |
| **Service ID** | Services identified by type (`DELIVERY`, `TAKEOUT`, `INDOOR`), without a separate ID |
| **`merchantType`** | Field removed |
| **`subtotal` on options** | Field removed. Use `option_price` (required) and `unity_price` |

### New modules

| Module | Description |
|---|---|
| **Customer / CRM** | Customer entity, leads, reviews — new 4th actor in the ecosystem |
| **Loyalty** | Points, cashback, coupons, and reward catalog |
| **Indoor** | Table, tab, and counter operations with central Account entity |

### Improvements

- **OAuth scopes by domain:** `od.orders` · `od.menu` · `od.logistics` · `od.crm` · `od.all`
- **Mandatory Discovery Well-Known:** `/.well-known/opendelivery` declaring capabilities before any integration
- **Status/event separation:** `status` on order GET; events are informational, not state definition
- **Authorization Code Flow:** optionally supported for advanced use cases
- **`quantity_available` on ItemOffer:** operational availability (not stock control)
- **Recursive options:** multiple levels via nested OptionGroups
- **Multiple services of the same type:** supported (was a V1 limitation)

### Not in RC (planned for V2.1)

- Reservations and waitlist (Indoor)
- Full LGPD / opt-in / opt-out detail
- Authorization Code Flow — full flow still being detailed
